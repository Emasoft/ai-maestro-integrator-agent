## Problem

The integrator's CI/release/publish gates fetch CPV with `uvx --from
git+https://github.com/Emasoft/claude-plugins-validation` (unpinned → resolves
to `master` HEAD). CPV released v2.137.0; its `[REPO LINT] (15 languages)`
phase regressed from ~18 s to a ~30-min hang on GitHub-hosted runners. Both
the `CI / Validate` job and the standalone `Plugin Validation` workflow timed
out (2/2) on v1.3.6 (commit `6eb9445`), while v1.3.5 (commit `6378fd1`) on
v2.136.1 was green in 74 s.

This is the downstream effect of the now-closed #139 (canon's broken `@main`
pin): the integrator had fallen back to a bare URL, which tracks LATEST and
therefore inherited the v2.137.0 regression.

## Decision

Pin every CPV callsite to the **`@v2.136.1` tag** — the last-known-good
release, verified green. This is canon's actual *intent* (version-pin for
stability), implemented with a real, working ref instead of the broken
`@main` or fragile bare master. A regressed/stricter future CPV release can no
longer break these gates with no plugin change; the pin is bumped deliberately
once CPV ships a fix for #148.

This composes with the integrator's existing local deviations (gitleaks
removed, no `CLAUDE_PRIVATE_USERNAMES`) which are version-independent, so
pinning to the pre-fix v2.136.1 costs nothing.
